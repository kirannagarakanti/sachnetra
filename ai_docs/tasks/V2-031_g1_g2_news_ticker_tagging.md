## The 10 Locked Decisions
*Resolved with Lijo (2026-05-26) after Gemini recon + codebase verification. Do not re-derive.*

### Decision 1 — Source: **NSE EQUITY_L.csv → committed `shared/nse-equity-master.json`**
`scripts/research/output/v2-031/nse_equity_master.csv` (2,386 rows, fetched 2026-05-26 with no cookie warm-up needed) is the seed. A build script transforms it to `shared/nse-equity-master.json` (committed). The tagger reads the JSON at module init — no runtime fetch, no warm-up dependency.

**Why this and not a runtime fetch:** the equity master changes only on listings/delistings (~weekly). Baking it in keeps the cron lean, removes one external failure point, and makes diffs visible in PRs. Refresh cadence is a future ops step: re-run `build-equity-master.mjs` weekly, commit the diff. Not V2-031 scope.

### Decision 2 — Tagger architecture: **dictionary-only Tier 1, no NER, no Python, no new deps**
The recon's NER eval (`ner_eval.csv`) confirmed that an extended equity master is the dominant component — generic NER models score high on CoNLL ORG F1 but underperform on Indian company names + cannot resolve brand aliases to NSE tickers at all. Dictionary catches the named entities the recon proved we need; NER would only add the long tail of typos/paraphrases.

**Tier 2 (spaCy `en_core_web_sm` fallback or HF async) is explicitly DEFERRED to V2-031b.** If Tier 1 coverage falls below the gate (Decision 7) on real prod data, V2-031b adds NER — but as a follow-up, not in this PR. Keeps V2-031 Node-only, zero new deps, ships in days not weeks.

### Decision 3 — Scope: **HEADLINE ONLY**
Body text doesn't exist anywhere in the pipeline (Context — Current State). V2-031 ships headline tagging. **V2-031b** (a separate task, not bundled) adds article-body fetch + an `article_body TEXT` column + a re-tag pass. Domain feasibility (paywall, anti-bot) is Lijo's probe at that point — not this task.

### Decision 4 — G2 piggyback: **bare NSE symbol at the source**
The new tagger emits **bare symbols** (`RELIANCE`, not `RELIANCE.NS`) because `shared/nse-equity-master.json` stores them bare (NSE's own EQUITY_L.csv uses bare symbols in the `SYMBOL` column). Nothing changes at `seed-india-signals.mjs:341` — the format change propagates upstream through `extractCompanies()`.

**Why this is the cleanest G2 fix:** single source of truth (the equity master). No `.replace('.NS','')` defensive strip anywhere; the format is correct from the start. `research_prices.symbol` keeps `.NS` (Yahoo requires it) — document that divergence in a comment at the migration site.

### Decision 5 — Alias generation: **suffix-strip cascade + manual brand overlay**
From `NAME OF COMPANY` (e.g. `"Page Industries Limited"`), the build script generates aliases by stripping suffixes in cascade (`["Page Industries Limited", "Page Industries", "Page"]`), each form added as an alias to that ticker. Suffix list locked: `Limited, Ltd, Pvt Limited, Private Limited, Corporation, Corp, Company, Co, Industries, Industry, Enterprises, Holdings, Group, International, (India), India` (the trailing `India` strip is skipped if it'd leave a generic word — Decision 6 handles the residual collisions).

**Brand overlay** (~30 entries, seeded by `alias_proposal.json`'s 12, extended by the build script — Jockey→PAGEIND, Domino's→JUBLFOOD, Maggi→NESTLEIND, Royal Enfield→EICHERMOT, etc.) is committed as a literal table at the top of `build-equity-master.mjs`. New brands can be added by editing that table and re-running the script. Not a runtime concern.

### Decision 6 — Disambiguation: **build-time collision filter + runtime context check (no runtime group defaults)**
Two distinct problems, two distinct mechanisms. Recon catalogued 20 such cases (`V2-031_g1_recon.md` §A3).

**(a) Multi-ticker collisions — filtered at BUILD time.** When the bare form (after the Decision 5 suffix-strip cascade) is shared by two or more tickers, the build script **emits neither as an alias**; both tickers keep only their multi-word forms. The runtime tagger never sees the ambiguous bare form, so no runtime disambiguation, ordering, or fallback logic is needed. Collisions are auto-detected by the build script (group by bare-form, drop where `count > 1`) and logged so the JSON diff in the PR shows exactly what got filtered.

Examples of bare forms expected to be filtered (non-exhaustive — the build script decides):
- `Apollo` (APOLLOHOSP / APOLLOTYRE / APOLLOMICRO) → only `Apollo Hospitals`, `Apollo Tyres`, `Apollo Micro` survive
- `Tata` (TCS / TATAMOTORS / TATASTEEL / TATAPOWER / TATAELXSI / …) → only the multi-word forms survive
- `Adani` (ADANIENT / ADANIPORTS / ADANIGREEN / ADANIPOWER / ADANITRANS) → only multi-word forms
- `Bajaj` (BAJFINANCE / BAJAJ-AUTO / BAJAJFINSV / BAJAJHLDNG) → only multi-word forms
- `JSW` (JSWSTEEL / JSWENERGY / JSWINFRA / JSWHL) → only multi-word forms
- `ICICI` (ICICIBANK / ICICIGI / ICICIPRULI) → only multi-word forms
- `Mahindra` (M&M / M&MFIN / MAHLIFE / MAHLOG) → only multi-word forms
- `Reliance` (RELIANCE / RPOWER / RELINFRA / RIIL / …) → only multi-word forms

**No runtime group-parent defaults.** Quant pipelines prefer precision over recall: false positives corrupt event-study joins (price moves attributed to the wrong ticker), while false negatives just leave a row untagged for V2-031b body-fetch. Losing bare-`Reliance` / bare-`Adani` headlines is acceptable at the 1.45% baseline — coverage headroom is huge.

**(b) Single-ticker common-noun collisions — RUNTIME context check.** Names that map to exactly one ticker but also look like English words (Lupin, Page, Titan, Britannia, Hero, Asian Paints) keep a per-entry "must-include-one-of" context word list, committed as an inline table in `build-equity-master.mjs` and serialized into `shared/nse-equity-master.json` as `denylist_context: [...]` per entry. The runtime tagger checks the context list for entries that have one; entries without it are matched plainly.

Examples:
- `LUPIN` requires one of: `pharma`, `drug`, `Q[1-4]`, `results`, `dividend`, `NSE`, `BSE`, `shares`, `share price`, `₹`
- `PAGEIND` (matched via `Page`) requires one of: `Industries`, `Jockey`, `innerwear`, `Q[1-4]`, `results`, `share`, `₹`
- `HEROMOTOCO` (matched via `Hero`) requires one of: `MotoCorp`, `motorcycle`, `two-wheeler`, `Q[1-4]`, `results`, `share`, `₹`

**Worked examples (the contract):**
- `"Tata Steel reports Q4 profit"` → multi-word `Tata Steel` matches → `['TATASTEEL']`
- `"Tata announces strategy"` → `Tata` not in alias map (collision-filtered) → `[]`
- `"Apollo Hospitals Q4 beats"` → multi-word `Apollo Hospitals` matches → `['APOLLOHOSP']`
- `"Apollo the spacecraft program"` → `Apollo` not in alias map → `[]`
- `"Reliance Industries Q4"` → multi-word `Reliance Industries` matches → `['RELIANCE']`
- `"Reliance announces buyback"` → `Reliance` not in alias map → `[]`
- `"Lupin Q4 pharma sales beat estimates"` → context word `pharma` present → `['LUPIN']`
- `"Lupin the gentleman thief returns"` → no context word → `[]`

**Follow-up (out of scope):** Once Lijo's sample verification exposes which filtered bare forms are costing real coverage (likely candidates: `Reliance` → RELIANCE, `Mahindra` → M&M, post-merger `HDFC` → HDFCBANK), a small allowlist of "dominant-in-news" single-word defaults can be added in V2-031c. **Not in V2-031.**

### Decision 7 — Coverage gate: **≥20% on new rows over a 48h window**
Lower than the historical 30% aspirational target. **Justification:** the recon's feed-mix analysis (~10 economy + ~35 politics + ~15 tech + ~3 disaster feeds) caps headline-only coverage at ~20–25%. 30% is achievable only after V2-031b (body fetch). The 20% gate is the proof-of-life threshold for Tier 1 — if we hit it, mid-cap tags are flowing in volume and the Exp 4/10 re-run can proceed; if we miss it, V2-031b is in scope sooner.

**Gate query** (the only number that matters):
```sql
SELECT
  100.0 * COUNT(*) FILTER (WHERE array_length(nse_tickers, 1) > 0) / COUNT(*) AS coverage_pct,
  COUNT(*) AS total_new_rows,
  COUNT(*) FILTER (WHERE array_length(nse_tickers, 1) > 0) AS tagged_new_rows
FROM india_news_signals
WHERE created_at > NOW() - INTERVAL '48 hours';
```

### Decision 8 — Backfill: **`nse_tickers_v2 TEXT[]` shadow column + verification window**
A new column `nse_tickers_v2 TEXT[] DEFAULT NULL` is added to `india_news_signals`. The backfill writes new tags into `nse_tickers_v2` for all 26,254 untagged rows. **Reads stay on `nse_tickers` throughout**, so live consumers see no change until the explicit cutover. After 24–48h verification (sample of 50 rows manually inspected by Lijo for per-source coverage), Lijo runs the cutover:
```sql
UPDATE india_news_signals SET nse_tickers = nse_tickers_v2 WHERE nse_tickers_v2 IS NOT NULL;
ALTER TABLE india_news_signals DROP COLUMN nse_tickers_v2;
```

**Why shadow column, not in-place rewrite:** a tagger bug that mis-tags 26k rows is unrecoverable without backups. Shadow column = trivial rollback (`DROP COLUMN`). Wall-clock estimate: ~5–15 minutes total for 26,254 rows × ~5ms tag + DB roundtrip (sequential batches of 500).

### Decision 9 — Forward-first rollout sequence
1. PR merges → Railway auto-deploys new tagger
2. Forward-tag flows live on the 10-min cron (new rows only — old rows still have `[]` or old `.NS`-suffixed tags)
3. **24–48h observation window** — Lijo runs the Decision-7 gate query against `created_at > NOW() - INTERVAL '48 hours'`
4. If ≥20% coverage: Lijo runs `backfill-news-tags.mjs` (re-tags all 26,254 untagged rows into `nse_tickers_v2`)
5. 24h verification: Lijo samples 50 rows from the backfill, compares `nse_tickers` vs `nse_tickers_v2` qualitatively
6. Lijo runs the cutover SQL (Decision 8) — `nse_tickers` is rewritten from `nse_tickers_v2`; shadow column dropped
7. Task closed. Exp 4 + Exp 10 mid-cap re-run is now unblocked.

**If gate misses (<20%):** STOP. Don't backfill. Diagnose with the per-source SQL (Decision 7 + recon E3) — likely cause is feed-mix (politics-heavy feeds drag the average down). Decide between expanding the alias list (cheap) and pulling V2-031b forward (NER fallback).

## Pipeline Flow

```
Build step (one-time, Lijo runs locally with the agent; commits the output):
  scripts/build-equity-master.mjs
    1. Read scripts/research/output/v2-031/nse_equity_master.csv  (committed; raw NSE master)
    2. Apply suffix-strip cascade (Decision 5) → generate aliases per ticker
    3. Overlay brand→corporate map (Decision 5, ~30 entries inline in the script)
    4. Attach disambiguation denylist (Decision 6, per-entry context words)
    5. Dedupe on SYMBOL (CSV has multiple rows per company for EQ/BE/GC series)
    6. Write shared/nse-equity-master.json    (committed)

Forward tagging (live, every 10-min cron, transparent change):
  scripts/_india-market-keywords.mjs::extractCompanies(title)
    1. At module init: load shared/nse-equity-master.json → build alias-to-ticker map
    2. Per call:
       a. Normalize title (lowercase, strip punctuation for matching)
       b. For each alias in the dictionary: word-boundary regex match
       c. If match AND ticker has a denylist entry: require one of the context words
       d. Collect matched tickers (bare symbols, no .NS)
       e. Return deduped array
  → consumed by seed-india-signals.mjs:341 unchanged; written to nse_tickers as bare strings

Backfill (one-time, Lijo runs after 48h forward observation):
  scripts/backfill-news-tags.mjs
    1. Loop until no rows returned:
         SELECT id, headline FROM india_news_signals
          WHERE nse_tickers_v2 IS NULL
          ORDER BY id
          LIMIT 500           -- NO OFFSET — UPDATEs below shrink the result set,
                              -- so each next SELECT naturally picks up the next batch
    2. For each row in the batch:
         - tags = extractCompanies(headline).map(c => c.ticker)
         - UPDATE india_news_signals SET nse_tickers_v2 = $1 WHERE id = $2
           (write '{}'::text[] for empty tags — NOT NULL — so the IS NULL guard
           advances and the loop terminates on no-tag rows)
    3. Log per-batch: batch N — processed 500, tagged K, cum total T (P%)
    4. Final log: total rows, total tagged, coverage %, wall-clock time

Cutover (Lijo, manual SQL after sample-verification):
  UPDATE india_news_signals SET nse_tickers = nse_tickers_v2 WHERE nse_tickers_v2 IS NOT NULL;
  ALTER TABLE india_news_signals DROP COLUMN nse_tickers_v2;
```

### A. NSE equity master
- **Source:** `https://archives.nseindia.com/content/equities/EQUITY_L.csv` — direct fetch, no cookie warm-up needed (verified by `scratch/fetch_nse_equity_master_playwright.mjs`).
- **Schema:** `SYMBOL, NAME OF COMPANY, SERIES, DATE OF LISTING, PAID UP VALUE, MARKET LOT, ISIN NUMBER, FACE VALUE`
- **Volume:** 2,386 rows (multiple `SERIES` rows per company — EQ, BE, GC — dedupe on `SYMBOL`).
- **Refresh cadence:** daily. Operational refresh strategy is out of scope for V2-031 — re-running `build-equity-master.mjs` weekly + committing the diff is the proposed pattern, but not part of this task.

### B. NER model evaluation
- Tested 5 candidates (spaCy sm, spaCy trf, dslim/bert-base-NER, Jean-Baptiste/roberta-large-ner-english, opennyaiorg/en_legal_ner_trf).
- **No Indian financial NER model exists on HuggingFace.** Closest is the legal-domain model — wrong domain.
- HF inference latency (200–500ms/call) + free-tier quota (30k calls/mo vs. 216k/mo needed) **both fail Tier 1 viability**. Conclusion locked in Decision 2: dictionary-only Tier 1; NER becomes V2-031b *if* Tier 1 misses the gate.
- Best ORG F1 available: `Jean-Baptiste/roberta-large-ner-english` at 96.44% on CoNLL-2003 — kept as the Tier 2 reference if V2-031b ships.

### C. Body-text availability (the gating question)
- ❌ NO body anywhere — `parseFeedItems()` extracts only `title` + `link`; no DB column; not in Redis digest. Confirmed from code in this codebase.
- V2-031 ships headline-only. V2-031b is the body-fetch follow-up.

### D. Brand-alias divergence
- Recon catalogued 12 confirmed brand→corporate mappings (Jockey→PAGEIND, Domino's→JUBLFOOD, Maggi→NESTLEIND, etc., full list in `alias_proposal.json`).
- **No GitHub repo provides this dataset for Indian equities.** Must be hand-curated. The 12-entry seed is committed; the build script extends to ~30 via the inline overlay (Decision 5).

### E. Disambiguation hazards
- 20 ambiguity cases catalogued (`V2-031_g1_recon.md` §A3): common-word collisions (Lupin, Apollo, Titan, Britannia, Asian Paints, Hero, Page), multi-subsidiary group names (Tata, Adani, Reliance, Mahindra, ICICI, Axis, JSW, Bajaj), and city-name collisions (Bank of Baroda, Bank of India).
- Denylist + context-word policy locked in Decision 6.

### F. G2 write paths (`scripts/research/output/v2-031/g2_write_paths.md`)
- `nse_tickers` is written in exactly **one place**: `seed-india-signals.mjs:341`, value sourced from `_india-market-keywords.mjs::extractCompanies()`.
- Changing the upstream output format (`extractCompanies` returns bare symbols) propagates G2 with zero edits at the write site.
- `research_prices.symbol` keeps `.NS` (Yahoo requirement); divergence documented.

## Pattern To Follow

Tagger module-init load + extract pattern (proposed shape; rewrite `_india-market-keywords.mjs`'s top + `extractCompanies` to this):

```javascript
import { readFileSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';

const __dirname = dirname(fileURLToPath(import.meta.url));

// Module-init load: built once per process, not per call.
const equityMaster = JSON.parse(
  readFileSync(join(__dirname, '..', 'shared', 'nse-equity-master.json'), 'utf8')
);

// Build alias→{ticker, name, denylist_context?} map at init.
// Per Decision 6(a), the build script has already collision-filtered ambiguous
// bare forms (Tata, Adani, Apollo, etc.) — duplicate aliases in the JSON would
// be a build-script bug, not a runtime concern.
const aliasMap = new Map();
for (const entry of equityMaster) {
  for (const alias of entry.aliases) {
    aliasMap.set(alias.toLowerCase(), {
      ticker: entry.ticker,
      name: entry.name,
      denylist_context: entry.denylist_context || null,
    });
  }
}

const ALIAS_REGEXES = [...aliasMap.keys()].map((alias) => ({
  alias,
  re: new RegExp(`\\b${alias.replace(/[.*+?^${}()|[\]\\]/g, '\\$&')}\\b`, 'i'),
}));

export function extractCompanies(title) {
  if (!title) return [];
  const lc = title.toLowerCase();
  // Dedup BY TICKER (Map<ticker, name>, not Set-of-objects — Set dedups by
  // object identity, which would let two aliases resolving to the same ticker
  // both slip through).
  const found = new Map();
  for (const { alias, re } of ALIAS_REGEXES) {
    if (!re.test(title)) continue;
    const entry = aliasMap.get(alias);
    // Decision 6(b): runtime context check for common-noun collisions
    // (Lupin, Page, Hero — single-ticker names that look like English words).
    if (entry.denylist_context) {
      const hasContext = entry.denylist_context.some((w) => lc.includes(w));
      if (!hasContext) continue;
    }
    if (!found.has(entry.ticker)) {
      found.set(entry.ticker, entry.name);
    }
  }
  // Return [{name, ticker}] — `seed-india-signals.mjs:341` reads `.ticker`
  // for the nse_tickers column, line 343 reads `.name` for the companies
  // column. Both must work post-change. `ticker` is BARE per Decision 4.
  return [...found].map(([ticker, name]) => ({ name, ticker }));
}
```

(Sketch — the shape is the contract. If `smoke-test-tagger.mjs` reports per-headline avg > 50ms, swap the per-alias loop for one combined-alternation regex per the Error Scenarios row. Do NOT pre-optimize.)

Backfill batch pattern (mirror `scripts/backfill-india-announcements.mjs` and `scripts/backfill-entity-timeline.mjs` — Node `pg.Pool`, `loadEnvFile`, sequential batches, idempotent `WHERE` guard).
