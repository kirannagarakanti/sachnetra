## Pattern To Follow

### JSON description parsing — from the render function in `setupDesktopIndiaLayout()`:

Currently, descriptions are rendered raw:
```typescript
<div class="sn-sidebar-finding-desc">${escapeHtml(f.description)}</div>
```

Add a sanitizer function BEFORE `escapeHtml()` that detects and parses JSON:
```typescript
const cleanDescription = (desc: string): string => {
  // If the description looks like JSON, try to extract summary text
  const trimmed = desc.trim();
  if (trimmed.startsWith('{') && trimmed.endsWith('}')) {
    try {
      const parsed = JSON.parse(trimmed);
      // Extract all .summary fields from the parsed object
      const summaries: string[] = [];
      for (const key of Object.keys(parsed)) {
        if (parsed[key]?.summary) summaries.push(parsed[key].summary);
      }
      if (summaries.length > 0) return summaries.join(' ');
    } catch { /* not valid JSON, use as-is */ }
  }
  return desc;
};
```

Then use: `escapeHtml(cleanDescription(f.description))`

## Session Notes — Lessons Learned for Future Agents

### Key decisions made

1. **Parse at display, not at source** — We fix the JSON at render time in the sidebar, not in the correlation service. This avoids touching the shared data pipeline and keeps the fix scoped to SachNetra.

2. **Generic JSON extractor** — The parser looks for any `.summary` field in the top-level keys of the JSON object. This handles both `summary1.summary` and `summary2.summary` patterns without hardcoding key names.
