## ⚠️ Read First — Three Locked Integration Decisions

The design doc's pipeline pseudo-code was written against the **pre-C4 V2-012 spec**
(per-new-cluster loop + `skipKnownClusters()`). **Shipped V2-012 is different** (the C4
two-tier redesign). These three decisions were resolved with Lijo and **override the
design doc wherever they conflict**:

### D1 — Linker integration point & idempotency key
There is **no "new clusters" set** and **no `skipKnownClusters()`** in shipped V2-012
(it became `getEnrichedClusterHashes()`). Therefore:

- Thread-linking runs **inside `fetchSignals()`**, **after** Tier 1 `persistSignals()`
  capture and **before** `buildDigest()`.
- It operates on captured clusters whose `india_news_signals` rows have **`thread_id IS NULL`**.
- **That NULL is the idempotency key.** Re-running on the same RSS state finds no
  unlinked clusters → attaches/spawns nothing. The design's "run twice → second run
  attaches nothing" criterion is re-expressed against `thread_id IS NULL`, **not** a
  48h skip window.
- R1–R6, the status sweep, the R6 `grownThreads` Set, and the 3rd Redis key
  (`news:threads:v1:india:en` via `runSeed` `extraKeys`) all stay exactly as the design
  doc specifies.

### D2 — Entity source = persisted columns, never re-extraction
The linker assembles each cluster's entity set, the rich `entities` object (Decision 5),
and `dominant_event_type` by **aggregating the cluster's already-persisted
`india_news_signals` row columns** — union of `nse_tickers` / `companies` / `sectors`
across the cluster's rows, mode of `event_type` across them. **No
`extractCompanies`/`extractSectors`/`detectEventType` re-run inside the linker** (avoids
extraction drift — the V2-014 soft-spot C lesson). `shared/market-taxonomy.json`
`nifty50_registry` (47 entries, shape `{ ticker, aliases:[...] }`) supplies canonical
ticker→name for the rich object; sector display names come from `taxonomy.sectors` keys.

### D3 — Tokenizer = export-only change to `_clustering.mjs` + sacred re-verify
Add `export` to the **existing** `tokenize()` and `jaccardSimilarity()` in
`scripts/_clustering.mjs`. **Purely additive — zero logic change, byte-identical
behavior.** `_thread-linker.mjs` imports them so threading and clustering tokenize
identically (R1). **Phase 6 mandatorily re-verifies `scripts/seed-insights.mjs` runs
unchanged after this export** (sacred-file bleed guard — `seed-insights.mjs` depends on
`_clustering.mjs`).

## Schema — Source of Truth

`story_threads` is **new** → `CREATE TABLE IF NOT EXISTS` is correct here. The FK on the
**existing populated** `india_news_signals` is **not** idempotent via `ADD CONSTRAINT`
(no `IF NOT EXISTS` in Postgres) — it MUST be guarded.

```sql
CREATE TABLE IF NOT EXISTS story_threads (
  thread_id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  thread_title         TEXT NOT NULL,
  thread_summary       TEXT,
  first_seen           TIMESTAMPTZ NOT NULL,
  last_seen            TIMESTAMPTZ NOT NULL,
  last_summary_at      TIMESTAMPTZ,
  status               TEXT NOT NULL,                 -- 'developing' | 'dormant' | 'resolved'
  event_count          INT NOT NULL DEFAULT 0,
  dominant_event_type  TEXT,                          -- R2: mode(event_type) over attached clusters
  entities             JSONB,                         -- Decision 5 rich object
  created_by           TEXT NOT NULL DEFAULT 'auto',  -- Decision 6
  created_at           TIMESTAMPTZ DEFAULT NOW()
);
CREATE INDEX IF NOT EXISTS idx_threads_status_last_seen ON story_threads (status, last_seen DESC);
CREATE INDEX IF NOT EXISTS idx_threads_last_seen        ON story_threads (last_seen DESC);
CREATE INDEX IF NOT EXISTS idx_threads_event_type       ON story_threads (dominant_event_type, status);

-- FK on existing populated table — guarded so re-running the migration is a no-op (C1 rule)
DO $$ BEGIN
  IF NOT EXISTS (SELECT 1 FROM pg_constraint WHERE conname = 'fk_signals_thread') THEN
    ALTER TABLE india_news_signals
      ADD CONSTRAINT fk_signals_thread
      FOREIGN KEY (thread_id) REFERENCES story_threads(thread_id);
  END IF;
END $$;
```

## Pattern To Follow

### Idempotent FK migration (C1 rule extended)
`CREATE TABLE`/`CREATE INDEX` use `IF NOT EXISTS`. The FK uses the guarded `DO $$` block
above. Append to the existing `DDL` string in `migrate-india-signals.mjs` — do not
restructure the file. Update its column-count log expectation only if it asserts a fixed
number (it logs the live count — no change needed).

### Reuse the Groq failover (no new call site)
`seed-india-signals.mjs` already has `doGroq(key, prompt-ish)` + `callGroqForCluster()`
implementing primary→`GROQ_API_KEY_2` failover on 429/5xx/network. For thread title-gen
and rolling re-summary, **reuse this path**: extract a small shared helper (e.g.
`callGroq(messages)` in `_seed-utils.mjs` or exported from `seed-india-signals.mjs`) and
have `_thread-linker.mjs` call it with its own prompt. Do not write a fresh
`fetch(GROQ_URL)` in the linker.

### Entity aggregation from persisted columns (D2)
After Tier 1 capture, for each cluster needing linking, `SELECT nse_tickers, sectors,
companies, event_type FROM india_news_signals WHERE cluster_hash = $1`. Union the arrays;
`dominant_event_type` = mode of `event_type` (alphabetical tie-break, design Open Q #7).
Map ticker→`{ticker,name,sectors}` via `nifty50_registry`; entities outside the registry
store `{ ticker: null, name, sectors }` (Decision 5).

### runSeed extraKeys — add a third entry
```js
extraKeys: [
  { key: DIGEST_KEY,  ttl: DIGEST_TTL,  transform: data => data.digest },
  { key: THREADS_KEY, ttl: THREADS_TTL, transform: data => data.threadsDigest },
]
```

## Error Scenarios

| Symptom | Likely cause | Fix |
|---|---|---|
| Migration fails 2nd run: "constraint already exists" | FK not wrapped in the `DO $$ pg_constraint` guard | Wrap per Schema block |
| `seed-insights.mjs` diff non-empty | `_clustering.mjs` change wasn't export-only | Revert; add ONLY the `export` keyword |
| Same cluster re-linked every run (double `event_count`) | Attach `UPDATE` missing `AND thread_id IS NULL`, or linker not scoped to unlinked clusters | Enforce D1 idempotency key everywhere |
| Threading degrades as thread ages | Jaccard vs short `thread_title` instead of title-bag | Implement R1 (union of last 10 cluster titles) |
| Earnings cluster joins disaster thread | R2 event-type guard missing | Skip candidate on `event_type` mismatch |
| Unrelated stories merged | R4 not filtering `common_entities` before overlap test | Load + apply `common_entities` |
| `thread_title` is a truncated headline | R5 cascade not reached on Groq failure | Cascade entity+event_type → entity → event_type event |
| 3 Groq summary calls for one thread in one cycle | R6 not batched | Defer to end-of-run `grownThreads` Set walk |
| `news:digest:v1:india:en` broke | Touched the existing digest path | Threads must be a separate key only (Decision 3) |
| Linker re-extracts entities and drifts | Called `extractCompanies` etc. | D2: aggregate persisted columns only |
