## The 6 Locked Decisions
*Forks resolved with Lijo (2026-05-19): FII-only via NSDL; cash-equity only.*

### Decision 1 — Source & access: **NSDL FPI Monitor `Archive.aspx`, ASP.NET handshake**
Endpoint: `https://www.fpi.nsdl.co.in/web/Reports/Archive.aspx`. Two-step:
1. **GET** `Archive.aspx` → collect `set-cookie` (take each cookie's `name=value`,
   join with `; `), and regex-extract hidden inputs `__VIEWSTATE`,
   `__VIEWSTATEGENERATOR`, `__EVENTVALIDATION` from the returned HTML.
2. **POST** `Archive.aspx` (form-urlencoded) with body:
   `__EVENTTARGET=btnSubmit1`, `__EVENTARGUMENT=`, the 3 hidden fields,
   `txtDate=<DD-MMM-YYYY>`, `hdnDate=<DD-MMM-YYYY>`, `hdnFlag=` — headers:
   Chrome UA, `Accept`, `Accept-Language`, `Referer: <Archive.aspx>`,
   `Origin: https://www.fpi.nsdl.co.in`,
   `Content-Type: application/x-www-form-urlencoded`, `Cookie: <joined>`.

**Why**: Confirmed working in `scratch/test_nsdl_post.mjs`. Public depository
data — **redistributable** (clears the Product C lineage flag for the historical
dataset). Date param format: **`DD-MMM-YYYY`** (e.g. `15-May-1999`).

**Implementation hint**: Re-fetch the GET (fresh `__VIEWSTATE`/cookies) if a POST
starts failing — ASP.NET viewstate/session can expire over a long walk. Success
markers in the POST HTML: contains `dvArchiveData` **and** `Reporting Date`.

### Decision 2 — Acquisition: **month-end walk, ~330 requests, polite pacing**
Querying any date returns that **whole month's daily rows up to that date**. So
query **one date per month** (month-end) from the earliest available month →
present. ≈ 12 requests/year × ~27 years ≈ **~330 requests total**.

**Why**: ~330 requests vs ~7,000 day-by-day. Confirmed by recon (the month-table
behaviour). Walk **sequentially with a small delay** (e.g. 1–2 s) — this is a
public regulator's site; do not hammer it. A 330-request polite walk is minutes.

**Implementation hint**: Earliest date is **to-confirm** — research summary says
daily series begins **1 Jan 1999**; `find_earliest_year.mjs` probes 1999 months
+ Dec 1998 and `test_nsdl_post.mjs` even probed 1997. Transcribe the agent's
actual earliest-`HAS DATA` month before locking the walk start (Research Appendix).

### Decision 3 — Target & idempotency: **existing table, `source='nsdl'`, supersede**
Upsert into `india_institutional_flows` (from V2-017) with
`investor_type='FII'`, `segment='cash'`, `source='nsdl'`, using V2-017's exact
`ON CONFLICT (flow_date, investor_type, segment) DO UPDATE` upsert.

**Why**: Zero schema change (V2-017 Decision 4 future-proofed this). `nsdl` is
**highest trust** (V2-017 trust order: `moneycontrol` < `nse`/`bse` < `nsdl`) —
so the NSDL retro-load **overwrites the overlapping ~30-day Moneycontrol window**
with depository-authoritative values. Idempotent → safe to re-run.

**Implementation hint**: Do NOT write a new upsert — import/copy V2-017's exact
upsert SQL so the trust-guard `WHERE` clause stays consistent.

### Decision 4 — Not a cron: **standalone one-time script, Lijo runs once**
`scripts/backfill-nsdl-fpi-history.mjs` is a plain script (no `runSeed`, no
Railway schedule, no Redis). Lijo runs it once against prod after review;
re-runnable to fill gaps.

**Why**: This is a historical backfill, not a recurring signal. V2-017's daily
Moneycontrol cron owns the forward edge. A cron here would be wasted machinery.

**Implementation hint**: Still `loadEnvFile()` for `DATABASE_URL`. Log progress
per year processed + a final `inserted/updated/skipped` summary. Exit non-zero
on a hard failure (this is a manual run, not a cron — failures should be visible).

### Decision 5 — Parse contract & resilience: **₹ crore, FPI-equity columns, skip-and-continue**
Parse the **2nd `<table>`** (index 1) of the POST HTML → rows → cells (strip
inner tags). Row 0 is the header. Map the **FPI equity gross-buy / gross-sell /
net** columns in **₹ crore** to V2-017's `gross_buy`/`gross_sell`/`net`.

**Why**: Matches `india_institutional_flows` (DECIMAL(14,2), ₹ crore). NSDL also
reports USD mn — **must select the ₹ crore columns**, not USD.

**Implementation hint**: Exact column order/labels are **to-confirm** from
`scratch/nsdl_post_response.html` / agent output before wiring the mapping
(Research Appendix has a placeholder). A month with no data (pre-listing,
holiday-only month) → log one line, continue the walk; never abort the whole run.

### Decision 6 — Prod-execution boundary
Per `memory/feedback_v2_prod_execution.md` + V2-017 Decision 6: deliver **code +
read-only self-checks only**. **Lijo runs** the backfill against prod after
review. The agent does NOT run the NSDL walk or create/run `scratch/` probes
(Gemini agent's job). Recon is delegated; capture + build is the agent's lane.

```

GET Archive.aspx → cookies + __VIEWSTATE/__VIEWSTATEGENERATOR/__EVENTVALIDATION
earliest = <confirmed earliest month>   # Research Appendix
for year in earliest..now:
  for month in 1..12 (skip future months):
    targetDate = `<monthEndDay>-<Mon>-<year>`   # DD-MMM-YYYY
    POST Archive.aspx (handshake body + txtDate/hdnDate=targetDate)
    if !(html has 'dvArchiveData' & 'Reporting Date'):
        log "[nsdl] no data <Mon-year>"; continue
    parse table[1] → rows[1:] → { reporting_date, fpi_equity_buy, sell, net (₹cr) }
    for each daily row:
        upsert india_institutional_flows
          (flow_date, 'FII', 'cash', buy, sell, net, 'nsdl', FALSE)   # V2-017 upsert
    sleep 1–2 s   # polite
    refresh GET handshake if POSTs start failing
log final: { months_walked, rows_inserted, rows_updated, months_empty }
```
