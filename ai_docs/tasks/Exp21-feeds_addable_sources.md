## ⚠️ Approach decision (latency-driven) — confirm before implementing

There are two ways to wire each source. **They are not equivalent for this experiment's purpose.**

| Approach | How | Pro | Con |
|---|---|---|---|
| **A — Direct RSS** *(recommended)* | native feed URL + domain in allowlist | **lowest latency** — preserves the head-start Exp21 is trying to measure | exact RSS path must be verified; some sites 403 from cloud IPs |
| B — Google-News proxy | `gnIn('site:moneycontrol.com')`, no allowlist change | robust, no 403 | **Google indexing delay (often 10–60+ min) destroys the head-start** — self-defeating for a latency source |

**Recommendation: Approach A (direct RSS) per source, with B as a per-source fallback only if the direct feed
403s from the Railway/Vercel egress IP.** Because the entire point of adding Moneycontrol is that it is *early*,
proxying it through Google News would erase the very signal we're adding it for.

**Candidate direct-RSS URLs — VERIFY each returns valid XML from the server egress IP before wiring** (do not
trust these blindly; confirm path + non-403):

| Source | Candidate RSS URL (verify) | Fallback if 403 |
|---|---|---|
| Moneycontrol | `https://www.moneycontrol.com/rss/latestnews.xml` (also `/rss/results.xml`, `/rss/business.xml`) | `gnIn('site:moneycontrol.com')` |
| Business Upturn | `https://www.businessupturn.com/feed/` (WordPress) | `gnIn('site:businessupturn.com')` |
| Free Press Journal | `https://www.freepressjournal.in/stories.rss` (confirm; try `/business/feed`) | `gnIn('site:freepressjournal.in')` |

If a source ends up on the `gnIn` fallback, **no allowlist entry is needed for it** (it goes through
`news.google.com`). Only direct-RSS sources need their domain added to the allowlist.

## Pattern To Follow

From `_feeds.ts`, `india.economy` mixes direct RSS and proxied entries — match this exact `ServerFeed` shape
(`{ name, url }`, optional `lang`); do not invent fields:

```typescript
economy: [
  { name: 'LiveMint', url: gnIn('site:livemint.com') },
  { name: 'Business Today', url: 'https://www.businesstoday.in/rssfeeds/?id=home' },
  { name: 'Hindu Business Line', url: 'https://www.thehindubusinessline.com/?service=rss' },
  // ...
],
```

Allowlist entries are bare hostnames, one per line, e.g. in `shared/rss-allowed-domains.json`:
```json
"www.business-standard.com",
"economictimes.indiatimes.com",
```

## Error Scenarios

| Symptom | Likely cause | Fix |
|---|---|---|
| 403 fetching the feed | Site blocks cloud egress IP | Switch that source to `gnIn('site:domain')`; remove its allowlist entry |
| Feed returns HTML, not XML | Wrong RSS path guessed | Find the correct RSS path on the site, or fall back to `gnIn` |
| Proxy 403 in browser fallback | Domain missing from one allowlist file | Add to **both** files (3-file rule); never edit `rss-proxy.js` |
| New feed adds no rows to `india_news_signals` | Separate "economy category" pipeline bug | Out of scope here — see V2 pipeline audit note above |
